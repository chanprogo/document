
## TCP 报文格式
　　　
TCP 报头中 的 
源端口号 和 目的端口号 
同 
IP 数据报中 的 源 IP 与 目的 IP   

唯一确定一条 TCP 连接。  

TCP 在发送数据前 必须 在彼此间 建立连接    


报文主要段的意思  
1. 序号：表示发送的 数据 字节流，确保 TCP 传输有序，对每个字节编号  
2. 确认序号：发送方 期待接收的 下一序列号，接收成功后 的数据字节序列号 加 1。只有 ACK=1 时才有效。  
3. ACK：确认序号 的标志，ACK=1 表示确认号有效，ACK=0 表示报文不含 确认序号 信息  
4. SYN：连接请求序号标志，用于建立连接，SYN=1 表示请求连接  
5. FIN：结束标志，用于 释放连接，为 1 表示 关闭本方数据流  






## 三次握手过程

建立 TCP 连接时，需要客户端和服务器共发送3个包。
• 第一次：客户端发送初始序号x和syn=1请求标志
• 第二次：服务器发送请求标志syn，发送确认标志ACK，发送自己的序号seq=y，发送客户端的确认序号ack=x+1
• 第三次：客户端发送ACK确认号，发送自己的序号seq=x+1，发送对方的确认号ack=y+1
